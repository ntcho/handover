프로젝트 디자인, 개발에 앞서서 저희가 개발하려는 앱, 프로그램에 대해서 조금 더 상의가 필요하다고 생각되어서 몇가지 토의가 필요하다고 생각한 점을 제시하려고 합니다.
1. 우리의 서비스를 사용할 사람들은 누구이고 어떤 환경일지        예를들어 "전입 신병, 신임 간부, 보직 변경" 의 예시가 있었는데 그렇다면 주 사용자가 누구인지, 사용자를 고려할 때 사용환경을 어떻게 생각하고 서비스를 개발 해야하는지,      군대내 컴퓨터나 핸드폰이 제한이 되는 환경은 어떻게 고려할 것인지 등등 다양한 상황이 있기 때문에 어떤 상황, 어떤 사용자를 중점으로 두고 서비스를 개발해야 되는지 토의가 필요하다고 생각합니다.
2. "어떤 인수인계 문서 형식이 가장 편리하고 효율적인지를 연구" 는 어떤식으로 진행할 계획이고 어떤 기준으로 선정할 것인지
3. 마지막으로 현행 인수인계 방법과 비교하여 저희가 개발하려는 서비스가 정확히 어떠한 장점을 가지고 있도록 할것인지 특히  어떠한 기능을 중심으로 둘 것인지에 따라 디자인이나 모티브가 될 서비스 설정이 많이 다를 것 같습니다
